### CLI Pattern with Typer
```python
# gaas_cli/main.py
import typer
from rich.console import Console
from rich.panel import Panel

app = typer.Typer(
    name="gaas-cli",
    help="Galaxy as a Service CLI",
    rich_help_panel="Main Commands"
)
console = Console()

@app.command()
def hello(
    name: str = typer.Argument(..., help="Your name"),
    formal: bool = typer.Option(False, "--formal", "-f", help="Use formal greeting"),
):
    """Say hello to someone."""
    greeting = f"Good day, {name}" if formal else f"Hello {name}"
    console.print(Panel(greeting, title="Greeting"))

if __name__ == "__main__":
    app()
```

### Rich Output Pattern
```python
from rich.console import Console
from rich.table import Table
from rich.panel import Panel
from rich.progress import Progress

console = Console()

def show_results(results: list[dict]):
    table = Table(title="Results")
    table.add_column("ID", style="cyan")
    table.add_column("Name", style="magenta")
    table.add_column("Status", style="green")

    for result in results:
        table.add_row(
            str(result["id"]),
            result["name"],
            "✓ Complete" if result["complete"] else "◌ Pending"
        )

    console.print(table)

def show_progress(items: list):
    with Progress() as progress:
        task = progress.add_task("Processing...", total=len(items))
        for item in items:
            # process item
            progress.advance(task)
```

### Data Processing with Pandas
```python
import pandas as pd
import pyarrow as pa

def process_data(df: pd.DataFrame) -> pd.DataFrame:
    """Process Galaxy analysis results."""
    return df.groupby("category").agg({
        "value": "mean",
        "count": "sum"
    }).reset_index()

def export_arrow(df: pd.DataFrame, path: str):
    """Export to Arrow format."""
    table = pa.Table.from_pandas(df)
    pa.parquet.write_table(table, path)
```

## Section 4: Testing Rules

### Test Commands
```bash
pytest                                      # Run all tests
pytest tests/                               # Specific directory
pytest -x                                   # Stop on first fail
pytest --lf                                 # Run last failures
pytest -k "test_import"                     # Filter by name
```

### Testing Standards
- **Framework**: pytest with pytest-cov for coverage
- **CLI Testing**: Use `typer.testing.CliRunner`
- **Mocking**: Use `unittest.mock` for external calls
- **Coverage**: Minimum 80% for commands
- **Fixtures**: Use `conftest.py` for shared fixtures

### CLI Test Pattern
```python
# tests/commands/test_hello.py
from typer.testing import CliRunner
from gaas_cli.main import app

runner = CliRunner()

def test_hello():
    result = runner.invoke(app, ["hello", "World"])
    assert result.exit_code == 0
    assert "Hello World" in result.output

def test_hello_formal():
    result = runner.invoke(app, ["hello", "World", "--formal"])
    assert result.exit_code == 0
    assert "Good day" in result.output
```

## Section 5: "Don't Touch" Zones and Permission Boundaries

### Protected Files
1. **src/gaas_cli/_version.py** - Auto-generated by hatch
2. **uv.lock** - Dependency lock file
3. **dist/** - Build artifacts (gitignored)

### Version Management
- Managed by `hatch` with `uv-dynamic-versioning`
- Pattern prefix: `gaas-cli-`
- Falls back to `0.0.0.dev0`
- Never edit `_version.py` manually

### Permission Boundaries
- **DO**: Create CLI commands for admin operations
- **DO**: Handle data export/import tasks
- **DO**: Integrate with Meilisearch for search operations
- **DO**: Process data with Pandas/PyArrow
- **DON'T**: Embed Galaxy API logic (use blendtype HTTP instead)
- **DON'T**: Access databases directly (use API endpoints)
- **DON'T**: Handle user authentication (done in Nuxt layer)

### Code Style
- **Formatter**: Black (line length 88)
- **Linter**: Ruff
- **Imports**: Sorted with `isort` (via ruff)
- **Type hints**: Required for public APIs
