This guide will walk you through setting up a new Nuxt project with GaaS.

## Prerequisites

Before starting, ensure you have:

- **Node.js** 18.0 or later (20.x recommended)
- **pnpm** 8.0 or later (recommended package manager)
- A **Galaxy instance** URL and API key

## Quick Start

::steps{level="4" title="Installation Steps"}

#### Create a Nuxt project

Create a new Nuxt project using the following command:

```bash [Terminal]
pnpm create nuxt <project-name>
```

During the creation process:
- Select the **Nuxt UI** module when prompted
- Choose your preferred options for TypeScript, linter, etc.

```bash [Terminal]
cd <project-name>
```

::note
The GaaS packages require Nuxt 4. The `create nuxt` command will scaffold a Nuxt 4 compatible project by default.
::

#### Clean up default files

Remove the default `app.vue` file so GaaS layers can provide their own:

```bash [Terminal]
rm app/app.vue
```

#### Install dependencies

Install the required dependencies for authentication and styling:

```bash [Terminal]
pnpm add @nuxtjs/supabase tailwindcss
```

#### Add GaaS package

Choose the GaaS package that fits your needs:

::tabs

:::tabs-item{label="GaaS UI (recommended)"}

For standard workflow interfaces:

```bash [Terminal]
pnpm add @gaas/ui
```

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  extends: ['@gaas/ui'],
})
```
:::

:::tabs-item{label="GaaS Wiki"}

For documentation-enabled workflow interfaces:

```bash [Terminal]
pnpm add @gaas/wiki
```

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  extends: ['@gaas/wiki'],
})
```
:::

::

#### Setup Tailwind CSS

Create `app/assets/css/main.css` with the following content:

```css [app/assets/css/main.css]
@import 'tailwindcss';
@import '@nuxt/ui';

@source "../../../node_modules/@gaas/ui";
```

#### Configure environment variables

Create a `.env` file in your project root:

```bash [.env]
# Supabase (we'll set these up in the next step)
SUPABASE_URL=""
SUPABASE_KEY=""
SUPABASE_SERVICE_KEY=""

# Galaxy
GALAXY_URL="https://your-galaxy-server.org"
GALAXY_API_KEY="<your-galaxy-api-key>"
GALAXY_EMAIL="<your-galaxy-email>"
```

::

## Next Steps

Your Nuxt project is now configured with GaaS. To complete the setup:

1. **Set up Supabase** - Follow the [Supabase Setup](/get-started/supabase-setup) guide to configure your database
2. **Configure Galaxy** - Update your `.env` with your Galaxy instance details
3. **Start developing** - Run `pnpm dev` to start the development server

## Verification

After completing all setup steps, start your development server:

```bash [Terminal]
pnpm dev
```

Your application should be running at `http://localhost:3000`. If you see the GaaS welcome page, everything is configured correctly!

## Troubleshooting

### Module not found errors

If you encounter "Cannot find module" errors, ensure:
- All dependencies are installed: `pnpm install`
- You're using the correct extends path in `nuxt.config.ts`
- Node.js version is 18+ (check with `node --version`)

### Styles not loading

If Tailwind styles aren't applying:
- Verify `app/assets/css/main.css` exists with the correct content
- Check that the `@source` path in the CSS file points to the correct node_modules location

### Build errors

If you get build errors after configuration changes:
```bash [Terminal]
pnpm nuxt cleanup
pnpm dev
```

## Existing Nuxt Projects

If you're adding GaaS to an existing Nuxt 3 project, you need to upgrade to Nuxt 4 compatibility:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  future: {
    compatibilityVersion: 4,
  },
})
```

Then run the migration codemod:

```bash [Terminal]
pnpm dlx codemod@latest nuxt/4/migration-recipe
```
