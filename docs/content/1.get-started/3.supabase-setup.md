title: Supabase Setup
description: Configure Supabase for your Nuxt Galaxy application.
---

This guide will walk you through setting up Supabase for your GaaS application, from installation to running migrations.

## Install Supabase CLI

First, install the Supabase CLI as a dev dependency in your project:

```bash [Terminal]
pnpm add -D supabase
```

::tip
Alternatively, you can install it globally with:
```bash [Terminal]
pnpm add -g supabase
```
::

## Initialize Supabase

Run the initialization command to create the `supabase/` directory with template configuration files:

```bash [Terminal]
pnpm supabase init
```

This creates a `supabase/` directory containing:
- `config.toml` - Main configuration file
- `migrations/` - Database migration files
- `seed.sql` - Seed data (optional)
- `.gitignore` - Preconfigured to ignore sensitive files

## Start Supabase

Start your local Supabase instance:

```bash [Terminal]
pnpm supabase start
```

This starts all Supabase services including PostgreSQL, GoTrue (auth), PostgREST (API), Realtime, and Storage. Once running, you can access:

- **Supabase Studio**: http://127.0.0.1:54323 (database management UI)
- **API**: http://127.0.0.1:54321

::tip
Add a convenience script to your `package.json`:
```json [package.json]
{
  "scripts": {
    "supabase:start": "supabase start",
    "supabase:stop": "supabase stop"
  }
}
```
::

## Run Migrations

GaaS provides a migration script that sets up the required database schema (tables, functions, and access controls) for the Galaxy integration. Run the migration command from your project root:

```bash [Terminal]
pnpm nuxt-galaxy-migrate
```

This command will:

- Create the `galaxy` and `galaxy_rbac` schemas
- Set up authentication tables and functions
- Configure storage buckets
- Apply Row Level Security (RLS) policies
- Enable Realtime for relevant tables

## Configure `config.toml`

After running the migrations, restart Supabase for configuration changes to take effect:

```bash [Terminal]
pnpm supabase stop
pnpm supabase start
```

Now modify the Supabase configuration file to expose the Galaxy schemas in the API. Open your `supabase/config.toml` and update the following sections:

### API Schemas

Enable the Galaxy schemas in the API:

```toml [config.toml]
[api]
schemas = [
  "public",
  "graphql_public",
  "galaxy",
  "galaxy_rbac",
  "storage"
]
extra_search_path = [
  "public",
  "extensions",
  "galaxy",
  "galaxy_rbac"
]
```

### Custom Access Token Hook

Enable the custom access token hook for role-based access control (RBAC):

```toml [config.toml]
[auth.hook.custom_access_token]
enabled = true
uri = "pg-functions://postgres/galaxy_rbac/custom_access_token_hook"
```

### Authentication Redirect URLs

Add your application's URLs to the allowed redirect locations:

```toml [config.toml]
[auth]
site_url = "http://localhost:3000"
additional_redirect_urls = [
  "http://localhost:3000",
  "http://127.0.0.1:3000",
  "http://localhost:3000/confirm",
  "http://127.0.0.1:3000/confirm",
  "http://localhost:3000/*",
  "http://127.0.0.1:3000/*",
]
```

### OAuth Providers (Optional)

If you want to enable OAuth authentication (e.g., GitHub), configure it here:

```toml [config.toml]
[auth.external.github]
enabled = true
client_id = "env(GITHUB_CLIENT_ID)"
secret = "env(GITHUB_CLIENT_SECRET)"
redirect_uri = "http://127.0.0.1:54321/auth/v1/callback"
```

::note
Make sure to add `GITHUB_CLIENT_ID` and `GITHUB_CLIENT_SECRET` to your environment variables.
::

## Environment Variables

Add the following environment variables to your `.env` file:

```bash [.env]
# Supabase
SUPABASE_URL="http://127.0.0.1:54321"
SUPABASE_KEY="<your-anon-key>"
SUPABASE_SERVICE_KEY="<your-service-role-key>"

# Galaxy
GALAXY_URL="https://your-galaxy-instance.org"
GALAXY_API_KEY="<your-galaxy-api-key>"

# OAuth (if enabled)
GITHUB_CLIENT_ID="<your-github-client-id>"
GITHUB_CLIENT_SECRET="<your-github-client-secret>"
```

::tip
For local development, the Supabase anon key is displayed in the terminal when you start Supabase. For production, use your project's API keys from the Supabase dashboard.
::

## Verify Setup

After starting Supabase, verify the setup by:

1. Opening Supabase Studio at http://127.0.0.1:54323
2. Checking that the `galaxy` and `galaxy_rbac` schemas exist in the database
3. Confirming that authentication functions are available

Your Supabase instance is now ready for GaaS development!
